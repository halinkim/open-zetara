이름 : zetara

## 앱 정보

- 기본적으로 pc 타겟 웹앱
- 오픈소스로, 사용자가 직접 빌드하거나 서버를 열 수 있음
- 데이터 백업이 쉬워야 함. 구독형이라고 하더라도 사용자가 여기에 데이터를 묶이는 느낌이 들지 않게.

- 사용하는 패키지 간의 충돌을 피하고, 부드럽고 완전한 기능 지원을 위해, 다른 라이브러리를 사용하지 않고 최소한의 라이브러리만 사용하여 최대한 독자적으로 기능 개발. (특히 캔버스 기능 등)


## 라이브러리

- 모든 개별 논문의 집합
- 기본적으로 zotero나 papers 와 같은 서지 관리 기능임
- 각 논문은 pdf가 거의 필수적임.
    - 저널, 제목, doi, 저자, url, pages, issue, volume, tag 등은 옵션
- pdf를 업로드(드래그앤 드랍 또는 직접 선택) 함으로써 하나의 논문 데이터가 생성됨.
- pdf의 메타데이터로부터 나머지 정보를 채움. doi등이 확정되면 crossref api 로 확인
- 물론 pdf가 없더라도 논문 데이터를 추가하는 것은 가능. 사용자가 직접 doi 등을 입력하고 패치 누르면 자동으로 메타 데이터 채워줌.
- 한 논문에는 여러 개의 pdf를 넣을 수 있음. supplement 자료 들을 추가로 넣는 것이 가능하다는 것. “논문 캔버스” 기능에서 여러 pdf가 있을 때 pdf간 전환을 지원해야 함. 대표 pdf는 첫 번째 pdf임. pdf 순서를 드래그로 변경할 수 있음.

- 논문의 export citation style을 지정할 수 있음. 다른 서지 관리 프로그램과 조금 다른 차별점인데, LIGO 논문처럼 저자가 매우 많을 경우, 저자를 모두 export하면 bibtex에서 너무 길어서 파싱오류로 안 됨. 이럴 때는 LVK Collaboration 처럼 실제로 표시될 이름을 지정해주거나 해야함.

- 다양한 검색 기능, 분류 기능을 제공함.
- 즐겨찾기, 태그, 캔버스가 없는 논문 보기 등.

### 참고문헌

- 각 논문을 불러올 때, crossref api로 참고문헌 정보도 함께 불러옴
- 각 참고문헌의 제목, 저자, 연도, 저널, doi 등의 정보를 패치하여 저장해둠
- 원하는 논문에 대해 이러한 참고문헌 목록을 확인하고, 사용자가 url 바로가기를 통해 해당 논문에 대한 pdf접근이 용이하도록 하여 새로운 논문을 추가할 때 편리성 제공
- 참고문헌은 doi로 구분함. 라이브러리 검색 기능에서 어떤 원하는 논문의 참고문헌을 검색할 수 있음. 이미 라이브러리에 존재하는 것과, 그렇지 않은 것으로 나뉘는데, 그렇지 않은 것은 바로 라이브러리에 추가할 수 있는 기능 제공함. pdf는 직접 넣어야 함.

## 논문 캔버스

- 각 논문에 대해 논문 캔버스를 통해 논문에 대한 다이어그램 등을 만들 수 있음
- 마치 논문에 대한 ppt 요약이라고 생각하면 편함
- 하지만 이 앱만의 특징이 여기서 나타남. 논문 pdf 로부터 이미지 형태로 직접 캡처하여 넣는 것이 아닌, 마치 포인터 형식으로, pdf의 영역을 선택하여 넣을 수 있음.
- 이렇게 넣어진 포인터 형식의 이미지는 클릭하면 해당 논문 pdf의 위치로 바로갈 수 있음(비슷한 기능이 일부 아이패드 노트 필기용 앱에도 있는 듯)
- 캔버스에서는 이러한 논문 부분 영역에 대한 이미지를 포인터 링크 형태로 넣을 수 있을 뿐만 아니라, 사용자의 임의의 이미지나 텍스트, 도형, 수식 등 ppt 편집하듯이 할 수 있음
- 레이아웃은 한 쪽에 pdf, 한 쪽에 캔버스가 있으며 오버리프나 vscode 등의 여러 편집기처럼 pdf만 나오게 보거나, 캔버스만 나오게 볼 수도 있음
- 각 논문에 대한 캔버스는 ppt와 같이 여러 장으로 구성될 수 있으며, 한 장의 사이즈는 무제한임. 따라서 원한다면 한 장 요약으로 할 수도 있음.

## 논문 그룹

- 라이브러리의 논문들로 구성된 논문 그룹을 만들 수 있음
- 각 논문은 여러 개의 그룹에 중복되어 존재할 수 있음
- 각 논문 그룹은 각 논문을 노드로 하여 시각화됨
- 논문 노드는 논문의 제목, 저자, 저널, 연도 등을 표시함.
- 논문 노드에서 논문 캔버스를 미리보기 형식으로 볼 수 있음
- 각 논문 노드들을 엣지로 이을 수 있음. 논문 노드의 상하좌우 부분에 이을 수 있음. 색, 두께, 화살표 스타일 등 커스터마이징 가능함.
- 논문 노드는 리사이징 가능함. 또한 논문 그룹에서 전체적인 패닝/줌이 가능함
- 논문 캔버스에서와 비슷하게 여기서도 사용자 커스텀 이미지나 텍스트, 도형, 수식 등을 넣을 수 있음.
- 따라서 사용자는 개별 논문 뿐만 아니라 논문들끼리의 관계까지 형성하면서 연구의 전체적인 맥락을 더 파악하기 쉬워짐.
